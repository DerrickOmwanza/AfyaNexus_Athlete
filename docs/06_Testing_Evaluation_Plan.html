1. Overview

This document outlines the testing and evaluation strategy for AfyaNexus. The plan covers all testing phases — unit, integration, system, user acceptance, performance, and security — to ensure the system meets its functional and non-functional requirements before final deployment and presentation.

5. User Acceptance Testing (UAT)

Purpose: Confirm the system meets the expectations and needs of real users.

5.1 UAT Participants

  • 2–3 Athletes (test morning check-in, training log, wearable sync, injury risk score)
  • 1–2 Coaches (test athlete list, performance overview, injury alerts)
  • 1 Nutritionist (test nutrition logs, diet plan creation, performance-nutrition chart)

5.2 UAT Scenarios

ScenarioUserStepsSuccess Criteria
Morning Check-In Athlete Log sleep hours, soreness, mood; view updated dashboard Data saved; dashboard reflects inputs within 2 seconds
Wearable Sync Athlete Connect T70 via Bluetooth; trigger sync; view data on dashboard Heart rate, sleep, steps appear on dashboard after sync
Injury Risk Review Athlete View injury risk gauge after logging data Risk score and level displayed clearly and accurately
Athlete Monitoring Coach View athlete list; open individual athlete dashboard; review alerts Only sponsored athletes visible; data accurate and up to date
Diet Plan Creation Nutritionist Open athlete nutrition dashboard; create and save diet plan Plan saved and visible on athlete's profile
Settings Update All Roles Navigate to Settings; update name and email; change password Profile saved successfully; password change takes effect immediately

UAT participants will complete a structured feedback questionnaire rating usability (1–5), data accuracy, and overall satisfaction. Results will be used to refine the UI and fix any identified issues before final submission.

8. Evaluation Metrics

ML Prediction Accuracy

Injury risk predictions compared against known outcomes from test datasets. Target: ≥ 80% accuracy.

Usability Score

Average UAT satisfaction rating across all user roles. Target: ≥ 4.0 / 5.0.

System Performance

Average dashboard load time and ML response time across all test scenarios. Target: Under 2 seconds and 3 seconds respectively.

System Reliability

System uptime during testing period. Target: ≥ 99%.

Security Compliance

Zero critical vulnerabilities detected across all security test cases.
